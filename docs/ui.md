# UI и TypeScript (app/static)

Документ описывает текущую реализацию UI после выноса из inline-HTML в статику и минимальной миграции на TypeScript.

## Структура файлов

- `app/static/index.html` — разметка страницы.
- `app/static/styles.css` — стили интерфейса.
- `app/static/ui.ts` — исходник клиентской логики.
- `app/static/ui.js` — скомпилированный JS, подключается в браузере.

FastAPI монтирует статику через `StaticFiles` и отдает UI на `GET /`.

## Сборка

Из корня проекта:

```bash
npm install
npm run build:ui
```

Команда `build:ui` компилирует `app/static/ui.ts` в `app/static/ui.js` через `tsc` (без SPA-бандлера).

## UX-поведение

- Валидация на клиенте: минимум 3 символа перед отправкой.
- Состояние загрузки: кнопка блокируется, отображается статус.
- Ошибки API и сети показываются в отдельном блоке ошибки.
- Результат содержит:
  - основной ответ;
  - список цитат (источник, title, короткий preview текста);
  - debug-блок в `details`.
- Горячая клавиша отправки: `Ctrl+Enter` / `Cmd+Enter`.

## Типы ответа API в UI

В `ui.ts` используется типизированный контракт:

- `QueryResponse` (`answer`, `citations`, `debug`);
- `Citation` (`id`, `source`, `title`, `text`);
- обработка `detail` как `string | object | array` для ошибок FastAPI.

## Ограничения текущего подхода

- Это не SPA и не компонентная архитектура: один модуль `ui.ts`.
- Нет frontend test runner (unit/e2e) в текущей конфигурации.
- Нет генерации TS-типов из OpenAPI (синхронизация пока ручная).

## Следующий шаг (Phase 3)
